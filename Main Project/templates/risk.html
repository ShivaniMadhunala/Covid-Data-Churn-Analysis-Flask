Insurance Risk Classification

For this project, the data comes from prudential life insurance competition on kaggle. The project solves the lengthy application process time and the goal for this project is to enhance the efficiency of processing time as well as reduce labor intensive tasks for classifying the new and existing customers based on how risky they are to the organisation.

Dataset Description

    Id: A unique identifier associated with an applicant.

    Product_Info_1-7: A set of normalised variables relating to the product applied for.

    Ins_Age: Normalised age of the applicant.

    Ht: Normalised height of the applicant.

    Wt: Normalised weight of the applicant.

    BMI: Normalised Body Mass Index of the applicant.

    Employment_Info_1-6: A set of normalised variables relating to the employment history of the applicant.

    Insured_Info_1-6: A set of normalised variables providing information about the applicant.

    Insurance_History_1-9: A set of normalised variables relating to the insurance history of the applicant.

    Family_Hist_1-5: A set of normalised variables relating to the family history of the applicant.

    Medical_History_1-41: A set of normalised variables relating to the medical history of the applicant.

    Mediacl_Keyword_1-48: A set of variables that indicate the presence of a disease ina an applicant.

    Response: The final result which gives the risk of the applicant from 1 to 8 where 8 being the highest risk.

Data Preprocessing

Stage 1: Creating a column which is the sum all the counts of the Medical History which gives the number of diseases the applicant has contracted.

Stage 2: Removal Of Columns with high missing values.
Columns Medical_History_10, Medical_History_24, Medical_History_32 and Family_History_5 are removed.

Stage 3: Processing Of Categorical Data by using One Hot Encoding.


Stage 4: Oversampling the data to make all the responses equal. Thus, improving efficiency of our code.

Results

We can observe that Random Forest Classifier has the highest around 78%. Thus we are able to classify the risk of a customer to the company and this solution can be scaled for many other real time applications.
